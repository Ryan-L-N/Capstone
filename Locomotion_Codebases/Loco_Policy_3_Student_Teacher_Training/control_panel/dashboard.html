Training Control Panel - User Guide

This dashboard lets you tune training parameters live without stopping the training run. Changes are picked up within 1 iteration (~0.3 seconds).

How It Works

The dashboard writes commands to a control.yaml file. The training loop polls this file every iteration. When it finds pending commands, it validates them through safety guardrails and applies them immediately to the running environment.

Browser (you)  -->  control.yaml  -->  training loop
  send command       pending_commands     poll_and_apply()
                          |                    |
                     atomic write          validates + applies
                     (no race)            reward_manager / LR / noise
                          |                    |
                          v                    v
                     status updated     change_log.jsonl

Controls Reference

Reward Weight Sliders

Each slider controls one reward term. Green values are task rewards (positive = more reward). Orange values are penalties (more negative = stronger penalty).

Drag a slider to adjust. The Send button appears when the value differs from the current training value. Click it to queue the change.

GuardrailRule
Sign PreservationRewards must stay positive, penalties must stay negative
Max DeltaChanges > 50% of current value are rejected (use CLI --force to override)
BoundsValues are clamped to safe ranges per reward term
Frozen Weightsstumble and body_height_tracking cannot be changed (Bug #22/#28b)

Obstacle Parkour Tuning Guide

The kinematic chain principle: tune rewards that govern the same physical motion together.

MotionReward TermsDirection for obstacle climbing
Step-up/overfoot_clearance, action_smoothness, joint_posfoot_clearance UP, action_smooth & joint_pos LESS penalty
Body tiltingbase_orientation, base_motionBoth LESS penalty (obstacles need body lean)
Power outputmotor_power, joint_torquesBoth LESS penalty (climbing needs force)
Gait qualitygait, air_time, air_time_varianceKeep stable or increase (prevents belly-crawl)

CLI Alternative

You can also use the CLI in a second SSH terminal:

cd ~/SIM_TO_REAL

# Show status
python -m control_panel.cli status

# Change weights
python -m control_panel.cli set action_smoothness -0.5
python -m control_panel.cli set foot_clearance 2.5 joint_pos -0.3

# Force past 50% delta guard
python -m control_panel.cli set joint_pos -0.1 --force

# Training params
python -m control_panel.cli lr --max 5e-4
python -m control_panel.cli noise --max 0.4

# S2R params
python -m control_panel.cli s2r max_dropout_rate 0.08

# Checkpoint + stop
python -m control_panel.cli save --label "before_experiment"
python -m control_panel.cli stop --reason "diverging"

# View history
python -m control_panel.cli history --last 30

Safety Notes

  • All changes are logged to control_panel_changes.jsonl with timestamps
  • Commands older than 120 seconds are automatically discarded
  • Emergency Stop saves a checkpoint before exiting
  • The dashboard auto-refreshes every 2 seconds
  • Multiple changes can be queued — they process in order next iteration
